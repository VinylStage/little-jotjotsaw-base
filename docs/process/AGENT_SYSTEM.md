# 에이전트 시스템 가이드

## 에이전트 조직도
```
Claude Code (Main Orchestrator)
└── Sub-agents (Search/Research, Code Review, etc.)
    └── VANCE (Local lightweight tasks)
        └── ORACLE (Heavy analysis, limited use)
```

## 각 계층의 역할과 사용 기준
- **Claude Code (메인 오케스트레이터)**  
  - 직접 처리: 단순한 코딩 작업, 직관적 판단이 필요한 경우  
  - 위임 기준: 서브에이전트/하위 작업이 필요하지 않은 경우 (예: 파일 생성, 간단한 테스트 코드 작성)

- **서브에이전트 (탐색/리서치, 코드리뷰 등)**  
  - 위임 기준: 도메인 전문성 필요 (예: 외부 API 문서 분석, 코드 리뷰)  
  - VANCE 대체 가능 여부 확인 후 호출 (예: "문서 초안 작성"은 VANCE로, "복합적 분석"은 서브에이전트로)

- **VANCE (로컬 경량 위임 작업)**  
  - 위임 기준: 반복적/가벼운 작업 (예: 문서 초안 작성, 간단한 데이터 정리)  
  - **제외 사항**: 복잡한 판단, 검증이 필요한 작업 (예: "이 코드가 보안 취약점이 있는지 분석" → ORACLE)

- **ORACLE (무거운 분석/검증)**  
  - 위임 기준: **절대적 제한**  
    - 복잡한 시스템 설계 검증 (예: "이 아키텍처가 확장성 문제 없을까?")  
    - **사전 점검 필수**: VANCE/서브에이전트로 해결 가능한지 확인 후 사용  
    - **사용 금지 사례**: 단순한 문장 수정, 반복적 데이터 변환

## ORACLE 호출 규칙
- **병렬 호출 금지**: 모든 ORACLE 호출은 **순차적 실행**으로 제한  
  - *사고 사례*: 과거 ORACLE을 동시에 여러 번 병렬 호출했다가 메모리 부족(OOM) 및 처리 지연/블로킹이 발생한 적이 있어, 이 규칙이 제정되었습니다.
- **리소스 확인 필수**:  
  - 호출 전 시스템/GPU 메모리 여유 확인 (`free -h`, `nvidia-smi` 등 환경에 맞는 도구 사용)
- **사용 후 정리**: ORACLE 호출 결과를 저장한 뒤, 불필요하게 프로세스를 띄워두지 않기

## 세션 위생
- **중복 세션 방지**:  
  - 새 Claude Code 세션을 시작하기 전, 동일 작업에 대해 이미 실행 중인 세션이 없는지 먼저 확인
  - "Claude Code 세션 중복 생성" 사고(2026년 7월)로 인해 **반드시 확인** 필요
- **세션 정리**:  
  - 작업 종료 후 더 이상 필요 없는 세션은 정리하여 리소스 낭비 방지

## 스케줄러 세션 제약사항 (2026-07-23 확인)

스케줄러 세션에서는 `mcp__dispatch__*` 도구(start_code_task, send_message 등)가 로드되지 않는다.
플랫폼 인프라 버그 (anthropics/claude-code#43397).

**워크어라운드**: 스케줄 태스크 프롬프트 첫 줄을 다음과 같이 작성:
> "직접 작업하지 말고 서브에이전트를 사용해서 전체 작업을 처리하라."

서브에이전트는 MCP 도구가 정상 초기화된다.

**대안**: bash로 파일 플래그 작성 후 외부 cron/launchd로 트리거.

## 예외 규칙: 기본 운영 룰 일시 해제 (2026-07-24 추가)

### 기본 원칙 (평상시)
- 루틴 코드 실행은 **로컬 LLM(opencode/aider via VANCE)**으로 위임 → Claude 토큰 절약
- Cowork(EVA)는 조율/보고 취합만, Code 세션(AXEL)도 직접 코딩 금지, opencode에 위임

### 예외 발동 조건

사용자가 **아래 중 하나의 신호**를 보낼 때:
- "이건 기존 룰에서 벗어나서 진행해줘"
- "예외 발동", "예외 처리", "이번만 예외"
- 기타 "기존 방식에서 벗어난다"는 뉘앙스가 명확한 경우

### 예외 발동 시 Claude의 행동 규칙

> **바로 진행하지 말고, 먼저 되묻기 필수**

예외 신호 수신 시 반드시 아래 내용을 확인 후 진행:

1. **어떤 룰을 예외 처리할 것인지** (예: "opencode 위임 룰", "단일 세션 룰", "토큰 절약 원칙" 등)
2. **예외의 범위와 기간** (예: "이번 작업만", "오늘 하루", "명시적으로 종료할 때까지")
3. **구체적으로 어떻게 진행할지** (예: "Cowork+Code 병행", "Claude가 직접 코딩", "복수 레포 동시 작업")

사용자 확인 후에만 예외 모드 진입. 추측으로 먼저 실행 금지.

### 예외 종료
- 해당 작업 완료 또는 사용자의 명시적 "종료/원래대로" 발언 시 기본 원칙 복귀
- 예외는 **1회성** — 다음 요청에서 자동 이월 없음
- 다음 대화/세션에서는 기본 원칙이 자동 복원됨

### 예외가 필요한 상황 예시
- 긴급 병행 작업 (다수 레포 동시 진행)
- 로컬 LLM 이슈로 opencode 사용 불가
- 사용자가 작업 속도를 토큰 비용보다 우선시
