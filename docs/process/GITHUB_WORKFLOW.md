# GitHub 워크플로 가이드

## 1. 계층 구조
GitHub의 계층 구조는 **Project → Milestone → Issue → Task**로 구성되며, 각 레포에서 공통으로 사용됩니다.

| 계층      | 설명                                                                 | 예시                                  |
|-----------|----------------------------------------------------------------------|---------------------------------------|
| **Project** | VinylStage 조직 전체를 아우르는 상위 보드 (다중 레포 공유)            | VinylStage의 공통 프로젝트 보드        |
| **Milestone** | 레포별 릴리스 단위 (버전명으로 명명, 예: `v0.3.0`)                    | `discord-mcp-alert` 레포의 `v0.3.0`    |
| **Issue**   | 기능/버그 단위 (제목에 `[feat]`, `[fix]` 등 커밋 규칙 적용)           | `[feat] 알림 시스템 추가`              |
| **Task**    | Issue 내 체크리스트 또는 하위 작업 (예: `[] API 연동 구현`)           | `[ ] Discord Webhook 통합`             |

---

## 2. 이슈 작성 규칙
- **제목 컨벤션**:  
  `[feat]` / `[fix]` / `[docs]` 접두사로 시작 (예: `[feat] 알림 메시지 필터링`).
- **최소 포함 항목**:
  1. **배경**: 왜 이 기능/버그가 필요한가? (예: "Discord 메시지가 과도하게 전송됨").
  2. **재현 방법 또는 요구사항**: 
     - 버그: "1. 알림 설정 → 2. 메시지 전송 → 3. 오류 발생".
     - 기능: "알림을 특정 채널로만 전송 가능해야 함".
  3. **완료 기준**: 
     - "알림 전송 시 채널 필터링 옵션 추가 완료".
     - "버그 발생 시 로그 출력 및 재시도 기능 구현".

---

## 3. release-please 운영 가이드
- **기본 조건**: 커밋 메시지에 **Conventional Commits** 규칙 적용 (예: `feat: 알림 시스템 추가`, `fix: 메시지 길이 제한 오류`).
- **자동화 흐름**:
  1. 커밋 메시지 규칙 준수 → `release-please`가 릴리스 PR 생성.
  2. PR 머지 → `CHANGELOG.md` 자동 갱신 (Features/Bug Fixes 분류, 커밋 링크 포함).
  3. 버전 태그 자동 생성 (예: `v0.1.0 → v0.1.1 → v0.2.0`).
- **실제 사례**:  
  `discord-mcp-alert` 레포에서 `v0.1.0` → `v0.1.1` → `v0.2.0`로 릴리스, 현재 `v0.3.0` 준비 중.
- **주의사항**:  
  릴리스 PR 머지 전 **로컬 브랜치와 원격 히스토리가 충돌**할 수 있음.  
  → **강제 push 대신 `git rebase`로 정리** (예: `git pull --rebase origin/main`).

---

## 4. Milestone 운영
- **방식**: 각 레포별로 **다음 릴리스 버전명**을 Milestone으로 생성 (예: `v0.3.0`).
- **작업 흐름**:
  1. 릴리스 준비 시 `v0.3.0` Milestone 생성.
  2. 관련 Issue를 Milestone에 연결.
  3. 릴리스 완료 후 Milestone 마감.

---

## 5. 새 프로젝트 레포 시작 시 GitHub 설정 순서
1. VinylStage 조직의 **Project**에 레포 연결.
2. 첫 릴리스 버전으로 **Milestone 생성** (예: `v0.1.0`).
3. `release-please` 설정 파일 추가 및 릴리스 규칙 적용.
4. **이슈 템플릿** 생성 (제목 컨벤션 및 필수 항목 포함).
