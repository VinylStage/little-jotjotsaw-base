# little-jotjotsaw-base — Claude 운영 가이드

## 1. 레포 역할 및 성격
이 레포는 VinylStage의 여러 사이드 프로젝트(예: finance-tracker, discord-mcp-alert 등) 공통 지식 자산을 저장하는 **코드 없음 지식 저장소**입니다. 프로세스 가이드, 트러블슈팅 로그, 에이전트 운영 가이드, 프로젝트 개요 문서만 관리되며, 다른 프로젝트 레포에서 작업 시 참조하거나 새로운 문서를 추가하는 용도로 사용됩니다.

## 2. Claude Code의 기본 태도
- **문서 정확성 최우선**: 기존 내용과 충돌하는 부분은 사람에게 즉시 확인 요청
- **추측 금지**: 구체적 근거 없이 내용을 추가하거나 수정하지 않기
- **사람 검수 의무**: VANCE 에이전트가 초안 작성 후 사람 검수를 반드시 거치기

## 3. 새 트러블슈팅 로그 추가 방법
- **파일명 규칙**: `docs/troubleshooting/YYYY-MM-프로젝트명.md`  
  (예: `2026-07-finance-tracker.md`)
- **기록 항목 필수 포함**:
  - 원인 분석
  - 재현 단계
  - 해결 절차
  - 재발 방지 대책
- **반드시 수행**:
  - `docs/troubleshooting/INDEX.md`에 새 파일 링크 추가
  - 파일명의 `프로젝트명`은 `docs/projects/` 하위 폴더명과 동일하게 사용

## 4. 새 프로젝트 개요 문서 추가 방법
- **파일 경로**: `docs/projects/<프로젝트명>/OVERVIEW.md`  
  (예: `docs/projects/discord-mcp-alert/OVERVIEW.md`)
- **필수 포함 항목**:
  - 사용 스택 (예: Python, Discord.py, PostgreSQL)
  - 아키텍처 한줄 요약 (예: "실시간 알림을 위한 웹훅 기반 메시지 처리 시스템")
  - 진행 단계 (예: 개발 완료, 테스트 중, 운영 중)

## 5. PROCESS_GUIDE.md 갱신 규칙
- **변경 승인 필수**: 기존 프로세스와 충돌하는 변경은 사람 승인 후만 반영
- **변경 이력 기록**: 파일 하단에 `YYYY-MM-DD: 변경 내용` 형식으로 추가
  (예: `2024-06-15: 트러블슈팅 로그 규칙 추가`)

## 6. 인덱스 파일 유지 방법
- **반드시 추가**:
  - 새 트러블슈팅 로그 → `docs/troubleshooting/INDEX.md`
  - 새 프로젝트 개요 → `docs/projects/INDEX.md` (추가 예정)
- **링크 형식**: `[YYYY-MM-프로젝트명](YYYY-MM-프로젝트명.md)`

## 7. 하지 말아야 할 것
- 다른 프로젝트의 기밀 정보(예: API 키, 정확한 금액, 사용자 데이터)를 그대로 복사하지 않기
- 레포 구조(폴더/파일명)를 임의로 변경하거나 삭제하지 않기
- 기존 문서 내용을 대체하거나 삭제하지 않기
- VANCE 에이전트가 초안 작성하지 않은 문서를 직접 작성하지 않기
