# little-jotjotsaw-base — Claude 운영 가이드

## 1. 레포 역할 및 성격
이 레포는 VinylStage의 여러 사이드 프로젝트가 공통으로 참조하는 **공통 지식/도구 저장소**입니다. 프로세스 가이드, 에이전트 운영 가이드, 교육용 프로세스 다이어그램, 그리고 여러 프로젝트에서 공용으로 쓰는 범용 도구(`tools/`)만 관리하며, 다른 프로젝트 레포에서 작업 시 참조하거나 새로운 공통 문서/도구를 추가하는 용도로 사용됩니다. **특정 프로젝트 전용 애플리케이션 코드, 특정 프로젝트의 이름, 트러블슈팅 로그, 프로젝트 개요 등 프로젝트별 내용은 이 레포에 절대 포함하지 않습니다** — 그런 내용은 각 프로젝트 자체 레포에 둡니다.

## 2. Claude Code의 기본 태도
- **문서 정확성 최우선**: 기존 내용과 충돌하는 부분은 사람에게 즉시 확인 요청
- **추측 금지**: 구체적 근거 없이 내용을 추가하거나 수정하지 않기
- **사람 검수 의무**: VANCE 에이전트가 초안 작성 후 사람 검수를 반드시 거치기
- **범위 준수**: 특정 프로젝트명이나 프로젝트별 세부사항이 포함된 내용을 요청받더라도 이 레포에는 작성하지 않고 사람에게 확인 요청

## 3. PROCESS_GUIDE.md 갱신 규칙
- **변경 승인 필수**: 기존 프로세스와 충돌하는 변경은 사람 승인 후만 반영
- **변경 이력 기록**: 파일 하단에 `YYYY-MM-DD: 변경 내용` 형식으로 추가
  (예: `2026-07-21: 우선순위 라벨 기준 추가`)

## 4. 새 다이어그램 추가 방법
- **파일 경로**: `docs/diagrams/주제명.md`
- **필수 포함**: Mermaid `flowchart` 코드블록 + 요약 설명
- **범위**: "이 레포 자체"의 공통 운영 프로세스만 다룰 것 (에이전트 조직, GitHub 워크플로, 세션 관리 등). 특정 프로젝트의 시스템 아키텍처는 다루지 않음.

## 5. 새 범용 도구 추가 방법
- **파일 경로**: `tools/<도구명>/` — 스크립트, `README.md`(사용법, 한국어), 필요 시 `.env.example` 포함
- **범위**: 특정 프로젝트에 종속되지 않고, `[repo]` 같은 파라미터나 자동 감지로 어떤 프로젝트 레포에서도 쓸 수 있어야 함. 특정 프로젝트 이름/경로를 하드코딩하지 않기
- **작성 방식**: 코드/스크립트는 VANCE 위임 대상이 아님 — 이미 검증된 코드를 이식하거나 직접 작성/검토할 것 (문서 초안만 VANCE에 위임)
- **문서 연계**: 관련 SDLC/프로세스 문서가 있으면 상호 링크 추가 (예: `docs/process/SDLC_WORKFLOW.md` ↔ `tools/<도구명>/README.md`)

## 6. 하지 말아야 할 것
- ❌ 특정 프로젝트 이름/트러블슈팅 로그/프로젝트 개요를 이 레포에 추가하지 않기
- ❌ 특정 프로젝트에만 쓸 수 있는 전용 로직이 담긴 스크립트를 추가하지 않기 (범용 도구만 허용)
- ❌ 다른 프로젝트의 기밀 정보(예: API 키, 정확한 금액, 사용자 데이터)를 그대로 복사하지 않기
- ❌ 레포 구조(폴더/파일명)를 임의로 변경하거나 삭제하지 않기
- ❌ 기존 문서 내용을 대체하거나 삭제하지 않기
- ❌ VANCE 에이전트가 초안 작성하지 않은 문서를 직접 작성하지 않기
